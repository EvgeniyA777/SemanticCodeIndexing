## Delta Classification

Fixture-by-fixture classification:

- `retrieval_elixir_aliased_local_ambiguity_001`: baseline debt, no tree-sitter-specific regression
- `retrieval_elixir_mixed_def_forms_001`: baseline debt, no tree-sitter-specific regression
- `retrieval_elixir_exunit_module_scenario_001`: baseline debt, no tree-sitter-specific regression

Observed conclusion:

- failing fixture set is identical across regex and tree-sitter
- failure-mode buckets are identical across regex and tree-sitter
- no tree-sitter-specific regression is currently evidenced by this subset

## Decision Impact

This note does not justify a remediation tranche by itself.

It supports the narrower conclusion:

- the current Elixir tree-sitter lane does not regress the recorded regex benchmark baseline on the existing Elixir fixture subset
- the benchmark subset is still red overall, so it also does not justify a default-switch approval

The next Elixir benchmark-oriented tranche should open only if one of the following becomes true:

- a new Elixir benchmark fixture passes in regex mode but fails in tree-sitter mode
- an existing shared-failure fixture develops a tree-sitter-specific failure mode
- maintainers choose to pay down the shared baseline debt regardless of parser mode
